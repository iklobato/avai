The problem

Your laptop is a black box.

You install a homebrew formula, a Cursor extension, a "free" PDF reader, an npm dependency. A week later something connects to a hostname you've never seen. You have no baseline to compare against — and no easy way to ask "is this normal?"

Commercial EDR is overkill, expensive, and ships your data to someone else. Native tools (ps, launchctl, systemctl, journalctl) are correct but tedious — by the time you've correlated them, the moment has passed.

The fix

A diff, judged by an LLM.

avai runs every 5 minutes, hashes every entity it finds, and only sends new things to the judge — so you spend tokens on signal, not noise.

Each finding comes back tagged malicious, suspicious, unknown, or benign — with a one-line remediation. The data never leaves your machine except the prompt itself, and the prompt only contains the artifact being judged.

What it sees

Twenty-one corners of your host. One screen.

processes / network

Every running PID, every open connection, every listening port. With the binary path, parent chain, and the user it runs as.

persistence

LaunchAgents / LaunchDaemons (macOS), systemd unit files + cron (Linux). The classic "what runs when I'm not looking" surface.

file integrity

/etc/passwd, /etc/shadow, sshd_config, sudoers, your shell dotfiles. Hashes every cycle; diff is loud.

hardware

USB devices (sysfs / system_profiler), paired Bluetooth, current Wi-Fi BSSID. Catches the "rubber ducky" and the rogue hotspot.

apps + extensions

Installed apps (dpkg / .app bundles), browser extensions across Chrome / Brave / Firefox / Edge. The most common malware vector still.

posture

SIP / Gatekeeper / FileVault / TCC.db (macOS) — SELinux / AppArmor / ufw / sshd / LUKS (Linux). Did something just downgrade your defaults?

streaming auth

A tail -f on every sshd, sudo, su, polkitd, systemd-logind event. Failed logins surface within seconds.

streaming exec

Every execve(2) on Linux (auditd), every process spawn on macOS (eslogger). Reverse shells don't get to live in peace.

setuid + mounts

Every +s binary on disk, every mounted filesystem with its options. Privilege-escalation primitives, on a list, judged.

The LLM judge

A second opinion at 3 AM, every 5 minutes.

avai uses litellm + Anthropic (Claude) to classify each new finding. Every prompt is structured: the collector, the entity, the judge fields. The model returns a verdict, a MITRE-aligned category, a confidence, and one line of remediation — no markdown, no "as a large language model" preamble.

  • → Dedup by content hash. Same artifact is never sent twice.
  • → Bring your own key — ANTHROPIC_API_KEY or CLAUDE_CODE_OAUTH_TOKEN.
  • → Or swap to any litellm-supported provider. Local Ollama works.
  • → Verdicts persist. Diff tells you what changed since last cycle.

Install

Three paths. Pick one.

Docker
recommended

Dashboard runs anywhere. Monitor needs a Linux host.

docker run -p 8765:8765 \
  -v "$PWD":/data \
  iklob1/avai
docker compose

Monitor + dashboard, both auto-restart.

curl -O https://raw.githubusercontent.com/\
iklobato/avai/main/docker-compose.yml
docker compose up -d
pip

Full macOS coverage needs the native install + sudo.

pip install 'avai[judge]'
sudo avai monitor &
avai dashboard
